Case study


'pyBreak' is a project created in goal to show my Python skills in areas of web scraping, data processing and analysis with properly prepared dataset. It's strictly connected with my hobbies - tennis is one of my favourite sports, which has many interesting aspects that are the subject of my data analysis.


Project presents the work performed in case of scraping two different data sources with help of Selenium library. Then, scraped data is being processed in Pandas. Both technologies are well-known for me due to my daily professional job. Further steps regarding the choice of specific dataset's parts for particular data analysis requires the ability to perform proper SQL queries (SQLite3 in this case), which results are shown by data visualization in popular Matplotlib.

Problem solving


  • Data comes from two different sources, which are needed to be parsed and standarized.
  • Standarization is needed due to different approach of presenting date formats, player's naming and matches scores' patterns.
  • Data should be processed in the way which allows to merge both sources into one project's base dataframe.
  • SQL and visualization skills are helpful to present interesting content behind the data.

Plans for the future relate to present the results of pyBreak backend job (which is done) in form of interactive dashboards here in the Django web application. Work is in progress.
